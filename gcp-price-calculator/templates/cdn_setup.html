How buckets work
SKU Patterns — one substring per line. A billing row is included if its SKU description contains any of these strings (case-insensitive).
Bucket Role — hit = cache egress data, miss = cache fill / downloads, lookup = request counts.
Lookup Divisor — for lookup buckets only. Usage amount is divided by this to get "units" (e.g. 10,000 = price per 10k requests).
Require list_cost > 0 — when on, free rows (list cost = 0) are excluded from this bucket. Recommended for hit/miss, off for lookups.
